Ring AllReduce — p 个GPU, 梯度 N 个元素.

分两阶段: Reduce-Scatter + AllGather.

Reduce-Scatter: 每GPU切 p 块, 每块 Np

step 0: GPUi 把 chunk (i−1)%p 发给 GPU(i+1)%p

step j (j=1..p−2): 收 → 累加 → 转发

共 p−1 步, 每步 Np, 完成后每GPU持有 1p 的全局和

AllGather: 同样 p−1 步, 每步 Np

完成后: 每GPU拥有完整 N 个归约结果

总通信量:

Tring = 2 · p − 1p · N

当 p 很大: T → 2N, 与 GPU 数无关 ← 带宽最优

延迟 = 2(p−1) · α ← 延迟不最优, 大 p 时 Tree 更好

α-β 通信模型:

T = 2(p−1) · α + 2 · p−1p · NB

α = 启动延迟, B = 链路带宽

N 大 → 带宽项主导 → Ring 最优; N 小 → 延迟项主导 → Tree 最优
