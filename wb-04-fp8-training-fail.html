FP8 E4M3 格式精度分析

IEEE-like 8位浮点: 1位符号 + 4位指数 + 3位尾数

尾数3位 → 隐含1 + 3位显式 = 4位有效精度

相邻可表示值的间距 = 2e−3

相对舍入误差:

εrel ≤ 2−3 = 0.125 = 12.5%

对比其他格式:

{

BF16: 尾数7位 → ε ≤ 2−7 = 0.78%

FP16: 尾数10位 → ε ≤ 2−10 = 0.098%

E4M3: 尾数3位 → ε ≤ 2−3 = 12.5%

E5M2: 尾数2位 → ε ≤ 2−2 = 25%

E2M1: 尾数1位 → ε ≤ 2−1 = 50%

最大可表示值:

E4M3: 448   E5M2: 57344   E2M1: 6

HYBRID策略: 前向 E4M3(精度优先) / 反向 E5M2(范围优先)

动机: 前向需要更精确的激活, 反向梯度值域更大

Attention 中 QKT 的值域分析

设 Q, K ∈ ℝN×d, 每个元素 ~ N(0, σ²)

则 S = QKT/√d 的每个元素:

Sij = 1√d ∑k=1..d Qik Kjk

由中心极限定理:

E[Sij] = 0

Var[Sij] = 1d · d · σ⁴ = σ⁴

→ Sij ~ N(0, σ⁴) (近似)

但训练后期, Q和K不再独立:

低秩表示涌现 → 某些注意力头中 Q ≈ K

此时 Sij 近似 ‖qi‖² cos(θ)√d

尾部极端值:

|Sij| 可达 4σ² ≈ 4√d · σ²/√d

训练后期 ‖q‖ 增长, 有效方差增大

关键洞察: 未缩放的 QKT 的值域:

|QKTij| ~ √d · |Sij| ← 可达 4√d · σ²

d=128, σ²=1 → |QKT| 可达 ~45

Softmax 中的 exp 溢出

标准 softmax: Pij = exp(Sij)∑k exp(Sik)

FlashAttention 用 exp2f 代替 exp:

exp(x) = exp2(x · log₂e) = exp2(x · 1.4427)

← 编译器可用 ffma 融合乘加, 节省1条指令

当 |QKT| 很大时:

d = 128 → |QKT| ~ 45

exp(45) = 3.49 × 1019

FP8 能表示吗?

E4M3 max = 448 → 溢出 (差 1017 倍)

E5M2 max = 57344 → 溢出 (差 1015 倍)

BF16 max = 3.39 × 1038 → 不溢出

根本矛盾:

FP8 省内存 ← 要存储中间结果 S, P

但 exp 输出的动态范围远超 FP8 能力

exp 的值域 [0, ∞) vs E4M3 的值域 [0, 448]

BF16 也不安全: 偏向舍入误差累积

BF16 加法中: 小数被大数"吞掉"

当 S 的行内方差大时 → softmax 分母的 偏向误差

ICLR'26 发现: 这是低精度训练爆炸的直接原因

解决方案 1: Softcap 限幅

核心思想: 用 tanh 将 QKT 限制在有限范围内

S̃ij = c · tanh(Sijc)

c = softcap 参数, 典型值 c = 30 或 50

性质:

当 |S| ≪ c:   S̃ ≈ S ← 小值不变

当 |S| ≫ c:   S̃ → ±c ← 大值被截断

值域: S̃ ∈ (−c, +c) ← 有界!

截断后 exp 是否安全?

c = 50 → exp(50) = 5.18 × 1021 → 仍然溢出 E4M3

c = 30 → exp(30) = 1.07 × 1013 → 仍然溢出 E4M3

但配合 online softmax 的 max 减去后:

exp(S̃ij − maxi) ∈ [0, 1] ← 安全

反向传播:

∂S̃∂S = 1 − tanh²(Sc)

当 |S| ≫ c: 梯度 → 0 ← 梯度消失

这限制了模型学习尖锐注意力分布的能力

Trade-off: 数值安全 ←→ 表达能力

Gemma-2 用 softcap=30, Claude 系列也使用类似策略

解决方案 2: 混合精度累加

核心思想: 计算用高精度, 存储用低精度

{

QKT 矩阵乘: FP8 输入, FP32 累加

softmax: FP32 全程

P·V 矩阵乘: FP8 输入, FP32 累加

存储 Q, K, V: FP8

内存节省分析 (N=seq_len, d=head_dim, h=heads):

标准 BF16: Q,K,V,O 各 2Nhd bytes

MBF16 = 4 × 2Nhd = 8Nhd

FP8 存储: Q,K,V,O 各 1Nhd bytes

MFP8 = 4 × 1Nhd = 4Nhd

节省: 50% 激活内存

split accumulator (Hopper架构):

硬件提供 FP8×FP8 → FP32 的 WGMMA 指令

但快速累加模式(fast_accum)截断中间结果到FP16

误差 = O(N · 2−10) ← N=seq_len 累积

split accumulator: 每 k 步做一次 FP32 规约

误差 = O(Nk · 2−10 + k · 2−23)

k 的选择: 小k → 精度好但慢, 大k → 快但精度差

失败模式: 注意力权重 P 的低秩退化

当多行 Q 相似 → S 的多行相似 → 舍入误差相干累积

随机误差 O(√N · ε) 变为系统性偏差 O(N · ε)

FP8 Delayed Scaling 的滞后效应

量化公式:

sf = FP8_MAXamax · 2−margin

FP8_MAX: E4M3→448, E5M2→57344

amax: 张量绝对值最大值

margin: 安全余量 (默认 0)

Delayed Scaling: 用上一步的 amax 计算本步的 sf

amax 历史窗口: H = 1024 步

选择策略: amax = max(H[0..1023]) 或 most_recent

4 种 edge case:

1 amax = 0 → scale 保持不变

2 0 < amax < tiny → scale → ∞ → 截断为 FP32_MAX

3 tiny ≤ amax < ∞ → 正常: sf = scaled_max / amax

4 amax = inf 或 NaN → scale 保持不变

滞后效应: 当张量值突变时

step t: amaxt = 10, sft = 448/10 = 44.8

step t+1: 真实 amax = 1000 (梯度突变)

但 sft+1 仍用 amaxt = 10 → sf = 44.8

量化: 1000 × 44.8 = 44800 > 448 → 饱和截断!

截断率:

P(clip) = P(|x| > FP8_MAXsf) = P(|x| > amaxold)

当分布突变: amaxnew ≫ amaxold → 大量截断 → loss spike

生产中观测: loss spike 与 amax 历史更新延迟高度相关

TAOCP 视角 — margin 最优值推导与批判

用户角度:

1 softcap 的 c 值选择无理论指导

c 太小 → 梯度消失, attention 分布模糊

c 太大 → 失去保护效果

最优 c*: 使 P(|S| > c) < δ, 即 c* = Φ−1(1−δ) · σS

但 σS 随训练动态变化 → 静态 c 必然次优

2 FP8 DPA 的精度-速度 Pareto 前沿未被探索

fp8_dpa=True 把 QKV 全转 FP8 → 速度 ↑

fp8_mha=True 连 LayerNorm 输出也保持 FP8 → 更快

但这两个 flag 的组合空间(4种)哪种最优? 无理论

系统角度:

3 margin 参数的最优值推导:

目标: 最小化总误差 = 量化噪声 + 截断损失

ε(m) = 2−(b+1)1 · 2m + Pclip(m) · E[|x|  |  clipped]

margin ↑ → 量化噪声 ↑ (分辨率降低)

margin ↓ → 截断概率 ↑ (更多值溢出)

对 m 求导 = 0 → m* = log₂(P'clip · E[|x|]ln2 · 2−(b+1))

默认 margin=0 意味着 NVIDIA 假设截断概率极低 ← 对稳定训练成立

4 跨硬件数值一致性:

NVIDIA FP8: per-tensor scaling, delayed

Trainium MXFP8: per-block scaling, block_size=32

相同超参 → 不同 loss 曲线 → 错误因果归因

5 低秩退化 + 偏向舍入 = 确定性崩溃

非随机故障, Knuth §4.2.2 的舍入分析直接适用:

N 次 BF16 加法的误差上界 = N · 2−8

当 N = seq_len = 8192: 偏差 = 32 ulp ← 灾难性

结论: FP8 训练不是"能不能用"而是"在什么条件下会崩"
